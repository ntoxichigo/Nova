Local-First Alpha

Agent chat first. Studio only when it helps.

Nova is a local-first agent chat workspace for technical users. The product story is intentionally narrow: start in chat, keep auth and API access locked down, and move into files, previews, or commands only when the task genuinely needs them.

Security posture
Remote UI is off by default. Remote API access requires `NOVA_API_SECRET`.
Primary audience
Technical alpha users who want an agentic chat surface without exposing every capability by default.
Release stance
Open-source alpha, biased toward clarity, guarded execution, and one hero workflow.
Hero Workflow

1. Start in chat

Define the task, keep the main surface simple, and let Nova plan or draft the first pass.

2. Move into Studio only when needed

Use files, previews, and commands as secondary tools when the work leaves pure conversation.

3. Keep the rest as support surfaces

Teach, Skills, Doctor, and Ops stay available, but they no longer compete with the main story.

Alpha Scope

What Nova is shipping now

  • Streaming agent chat with guarded tool execution
  • Local Studio for project-aware files, previews, and commands
  • Memory, teach, and skill support for iterative technical work
  • Provider support for local and hosted LLMs

What we intentionally de-emphasize

  • Surface sprawl that distracts from the chat-first experience
  • Remote-by-default exposure
  • Over-marketing broad capabilities before the core loop feels sharp

Who this is for

  • Developers who want a local-first agent workspace
  • Technical users comfortable running an alpha
  • Contributors who want a focused open-source direction

Quick start

  1. Install dependencies
  2. Copy `.env.example` to `.env`
  3. Set `DATABASE_URL` and `TOKEN_ENCRYPTION_SECRET`
  4. Generate the Prisma client and push the DB
  5. Run the dev server

Open-source alpha notes

The repository is being prepared as an Apache 2.0 open-source alpha. SQLite works locally; PostgreSQL is preferred for production-style deployments.

Nova is a focused alpha now.

The goal is not to sell every surface at once. The goal is to make one strong workflow obvious, trustworthy, and easy for technical users to adopt.
