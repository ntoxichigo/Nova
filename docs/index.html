Local-First Alpha

Nova turns messy technical requests into a clear execution loop.

Nova is a local-first agent chat workspace for developers and technical operators. Use it to diagnose a bug, shape a rollout, inspect a codebase, draft changes, and then widen into files, previews, and commands only when the work becomes concrete.

  • Start narrow: define the task in chat and keep the first interaction focused.
  • Go deeper when needed: open Studio for real files, previews, and command execution.
  • Stay guarded: remote access is not the default story, and auth stays locked down.

Hero Workflow

The product story is intentionally narrow.

Nova is strongest when the flow is obvious: start in chat, then widen only when the work becomes operational.

1. Start in chat

1
Define the task, ask for diagnosis, or request a first-pass plan.
This is where ambiguity gets compressed into something workable.
2
Let Nova reason, summarize, compare options, or propose implementation steps.
You should not have to jump into a full IDE surface just to get clarity.

2. Escalate only when the task becomes concrete

3
Open Studio for files, previews, commands, or generated project structure.
The tools support the conversation instead of competing with it.
4
Use Teach, Skills, Doctor, and Ops as secondary surfaces.
Helpful, but not the first thing a new user has to understand.
Capabilities

What Nova can actually help you achieve.

Not vague “AI magic.” These are the concrete classes of work the product is already shaped around.

Debug and unblock technical work

Trace failing flows, summarize where the issue probably lives, and turn scattered context into a concrete next action.

Plan implementation before editing

Break a feature or cleanup into steps, tradeoffs, and likely file touch points before you start making changes.

Generate and shape project scaffolds

Use the IDE side when you need generated files, previews, or an editable starting point for a concrete build.

Accumulate working knowledge

Teach Nova recurring facts, preferences, and reusable instructions so later conversations start from a better place.

Route work through specialist surfaces

Memory, Skills, Doctor, and Ops can support advanced workflows without replacing the core chat mental model.

Keep remote exposure under control

Nova is framed for local-first technical use, with remote UI and API exposure deliberately not treated as the default.

Docs And Setup

Everything you need to evaluate the alpha.

The setup story is still practical and developer-first: install, configure local secrets, initialize the database, and run.

Quick start

  • `npm install`
  • Copy `.env.example` to `.env`
  • Set `DATABASE_URL` and `TOKEN_ENCRYPTION_SECRET`
  • Run `npm run db:generate` and `npm run db:push`
  • Start with `npm run dev`
